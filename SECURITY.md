# Security Policy

## Supported Branches

Currently, only the `master` branch is actively supported with security updates and bug fixes. Please make sure your contributions or usage are based on this branch.

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it immediately via email to **[isherezahin@gmail.com](mailto:isherezahin@gmail.com)**. Do not open an issue publicly.

All security reports will be handled **confidentially** and **promptly**.

## Responsible Disclosure

* Please provide a detailed description of the issue.
* Include steps to reproduce the vulnerability if possible.
* Do not exploit the vulnerability yourself.
* Keep all details confidential until the issue has been resolved.

## Important Security Notes

* Do not share any API keys, passwords, or sensitive data publicly.
* Always ensure sensitive information is kept private and secure.
* Regularly update dependencies to minimize security risks.

## Response

The maintainer will acknowledge your report as soon as possible and communicate any updates on the resolution.

Thank you for helping keep this project safe.
